## 二、PostgreSQL 表结构

### 2.1 管理员用户表 `users`

```sql
CREATE TABLE users (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username    VARCHAR(50)  NOT NULL UNIQUE,
    password    VARCHAR(255) NOT NULL,  -- bcrypt hash
    email       VARCHAR(255),
    role        VARCHAR(20)  NOT NULL DEFAULT 'admin',  -- 'admin' | 'super_admin'
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

### 2.2 对话表 `conversations`

```sql
CREATE TABLE conversations (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id  VARCHAR(100) NOT NULL,  -- 匿名用户会话标识
    title       VARCHAR(200) NOT NULL DEFAULT '新对话',
    message_count INT        NOT NULL DEFAULT 0,
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_conversations_session_id ON conversations(session_id);
CREATE INDEX idx_conversations_updated_at ON conversations(updated_at DESC);
```

### 2.3 消息表 `messages`

```sql
CREATE TABLE messages (
    id              UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id UUID         NOT NULL REFERENCES conversations(id) ON DELETE CASCADE,
    role            VARCHAR(20)  NOT NULL,  -- 'user' | 'assistant' | 'system'
    content         TEXT         NOT NULL,
    token_count     INT,                    -- 该消息的token数
    created_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_messages_conversation_id ON messages(conversation_id);
CREATE INDEX idx_messages_created_at ON messages(created_at);
```

### 2.4 反馈表 `feedbacks`

```sql
CREATE TABLE feedbacks (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    message_id  UUID        NOT NULL REFERENCES messages(id) ON DELETE CASCADE,
    rating      VARCHAR(10) NOT NULL,  -- 'like' | 'dislike'
    reason      VARCHAR(100),          -- 'inaccurate' | 'incomplete' | 'irrelevant' | 'other'
    comment     TEXT,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_feedbacks_message_id ON feedbacks(message_id);
```

### 2.5 知识库文档表 `documents`

```sql
CREATE TABLE documents (
    id                UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    filename          VARCHAR(500) NOT NULL,
    original_name     VARCHAR(500) NOT NULL,
    file_type         VARCHAR(20)  NOT NULL,  -- 'pdf' | 'docx' | 'md' | 'txt' | 'url'
    file_size         BIGINT,                 -- 字节数
    chunk_count       INT          NOT NULL DEFAULT 0,
    char_count        INT          NOT NULL DEFAULT 0,
    status            VARCHAR(30)  NOT NULL DEFAULT 'uploading',
    -- uploading | parsing | indexing | completed | failed
    error_message     TEXT,
    source_url        VARCHAR(1000),          -- 来源URL（如适用）
    category          VARCHAR(100),           -- 分类：新生指南/教务制度/...
    uploaded_by       UUID REFERENCES users(id),
    created_at        TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at        TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_documents_status ON documents(status);
CREATE INDEX idx_documents_category ON documents(category);
```

### 2.6 埋点事件表 `analytics_events`

```sql
CREATE TABLE analytics_events (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id  VARCHAR(100) NOT NULL,
    event_type  VARCHAR(50)  NOT NULL,
    -- 'page_view' | 'question_asked' | 'feedback_given' | 'quick_entry_click'
    event_data  JSONB,                  -- 事件附加数据
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_analytics_events_type ON analytics_events(event_type);
CREATE INDEX idx_analytics_events_created_at ON analytics_events(created_at);
CREATE INDEX idx_analytics_events_session ON analytics_events(session_id);
```

## 三、ChromaDB Collection 设计

### 3.1 知识库 Collection：`campus_knowledge`

```yaml
collection_name: campus_knowledge
embedding_model: text-embedding-3-small (OpenAI) 或 bge-large-zh-v1.5 (本地)

metadata_fields:
  - doc_id: string       # 对应 documents 表的 id
  - filename: string     # 来源文件名
  - category: string     # 分类（新生指南/教务制度/...）
  - chunk_index: int     # 分段序号
  - source_url: string   # 来源URL（可选）
  - updated_at: string   # 更新时间
```

### 3.2 分段策略

```yaml
chunk_size: 500 tokens      # 每段约500个token
chunk_overlap: 50 tokens    # 段间重叠50个token
separator: "\n\n"           # 优先按段落分割

embedding_batch_size: 20    # 每批20个chunk调用embedding API
```

## 五、索引策略

| 表 | 索引字段 | 原因 |
|----|---------|------|
| conversations | session_id | 按用户查询对话 |
| conversations | updated_at DESC | 对话列表排序 |
| messages | conversation_id | 按对话查消息 |
| messages | created_at | 时间范围查询 |
| feedbacks | message_id | 按消息查反馈 |
| documents | status | 按状态筛选 |
| documents | category | 按分类筛选 |
| analytics_events | event_type + created_at | 分析查询核心 |
| analytics_events | session_id | 用户行为分析 |

## 六、数据安全

- 密码使用**bcrypt**（salt rounds≥12）加密存储
- 对话数据**不关联**用户身份，仅用 session_id 标识
- 系统不收集任何个人身份信息（PII）
- 日志中不记录消息内容
- 数据库备份：每日全量 + 每小时增量
- 对话数据保留策略：6个月后自动归档清理

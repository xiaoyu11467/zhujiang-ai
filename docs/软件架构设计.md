## 二、技术栈

### 前端
| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | React 18+ | 函数组件 + Hooks |
| 语言 | TypeScript | 全量类型定义 |
| 样式 | Tailwind CSS 3+ | 原子化CSS + 自定义主题 |
| 路由 | React Router v6 | SPA路由 |
| 状态管理 | Zustand | 轻量级状态管理 |
| HTTP | Axios | API请求 + 拦截器 |
| Markdown | react-markdown | AI回答MD渲染 |
| 聊天UI | 自研组件 | 对话气泡、输入框、侧边栏 |
| 图表 | Recharts | 后台数据可视化 |
| 构建 | Vite | 快速开发与构建 |

### 后端
| 层 | 技术 | 说明 |
|----|------|------|
| 框架 | NestJS | 模块化Node.js框架 |
| 语言 | TypeScript | 全量类型定义 |
| ORM | Prisma | 数据库ORM |
| 数据库 | PostgreSQL 15+ | 业务数据存储 |
| 向量数据库 | ChromaDB | RAG知识检索 |
| 文档解析 | pdf-parse + mammoth | PDF/Word解析 |
| 网页抓取 | cheerio | HTML内容提取 |
| 认证 | JWT + bcrypt | 管理员登录 |
| API文档 | Swagger | 自动生成API文档 |
| 验证 | class-validator | DTO数据验证 |

### AI
| 层 | 技术 | 说明 |
|----|------|------|
| LLM | DeepSeek API | 主要模型（性价比高） |
| LLM备用 | OpenAI API | 兼容接口 |
| Embedding | text-embedding-3-small / bge-large-zh | 中文向量化 |
| RAG框架 | LangChain (JS版) | 检索增强生成 |
| 提示词 | 自建Prompt模板 | 校园助手专有 |

## 四、数据流

### 4.1 AI问答流程

```
用户输入问题
     │
     ▼
┌─────────────────┐
│  ChatController  │  POST /api/chat/send
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   ChatService    │  创建对话/消息记录
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    RAGService    │  知识检索
│                 │
│ 1. 问题Embedding │
│ 2. ChromaDB搜索  │
│ 3. 返回Top-K文档 │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│    LLMService    │  生成回答
│                 │
│ 1. 组装Prompt    │
│ 2. 调用LLM API   │
│ 3. 返回流式响应   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  SSE Stream      │  流式返回前端
│  Server-Sent     │
│  Events          │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   前端渲染        │  逐字显示 + Markdown
└─────────────────┘
```

### 4.2 文档入库流程

```
上传文件 (PDF/Word/MD/TXT)
     │
     ▼
┌─────────────────┐
│  文件类型判断     │
└────────┬────────┘
         │
         ├─── PDF → pdf-parse 提取文本
         ├─── Word → mammoth 提取文本
         └─── MD/TXT → 直接读取
         │
         ▼
┌─────────────────┐
│   文本分块        │
│  (Chunk Size:    │
│   500 tokens)    │
│  (Overlap: 50)   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Embedding      │  每个Chunk向量化
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   ChromaDB       │  存储向量+元数据
│   入库完成        │
└─────────────────┘
         │
         ▼
┌─────────────────┐
│   PostgreSQL     │  记录文档元信息
└─────────────────┘
```

## 七、安全设计

- **认证**：JWT Token + 刷新Token机制
- **限流**：@nestjs/throttler 限制API调用频率
- **CORS**：配置允许的前端域名
- **输入校验**：class-validator 验证所有输入
- **SQL注入防护**：Prisma ORM参数化查询
- **XSS防护**：前端输出转义
- **文件上传**：限制文件类型和大小（最大10MB）
- **环境变量**：敏感信息通过.env管理，不提交Git
